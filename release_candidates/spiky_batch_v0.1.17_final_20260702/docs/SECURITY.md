# Security Policy

Report ordinary, non-sensitive defects through GitHub Issues and include only synthetic or public-safe reproductions.

Do not post credentials, private paths, personal information, unpublished experimental data, or security-sensitive details publicly. This project does not yet publish a dedicated private security contact. If GitHub's private vulnerability reporting is enabled, use it; otherwise contact the repository owner through an available private GitHub channel before sharing details. If no private channel is available, report only that a sensitive issue exists and wait for private contact instructions.
